Test Taker Interaction Overview

  • Test taker ability is fixed but unknown: $\theta \sim p(\theta)$
  • Questions generated from latent variable $z \sim p(z)$: $q \in \mathcal{Q}$
  • Response Model (Rasch’s Model)

    $$p(y = 1 \mid z; \theta) = \sigma(\theta - z)$$

Estimating Ability Efficiently

  • Goal: Estimate $\theta$ with a budget of $K$ questions
  • Adaptive Question Selection
    • Guided by acquisition functions like Fisher Information
    • Use empirical reliability $\mathcal{R}$ and MSE to evaluate accuracy

Fisher Information and SEM

  • Fisher Information:

    $$\mathcal{I}(\theta) = \sum_{i=1} p_i(1 - p_i), \quad \text{where} \ p_i = p(y = 1 \mid \theta, z_i)$$

  • Standard Error of Measurement (SEM)
    • Defined as the square root of the inverse Fisher Information

Empirical Reliability & MSE

  • Empirical Reliability $\mathcal{R}(\theta)$

    $$\mathcal{R}(\theta) = 1 - \frac{\sum \text{SEM}(\theta_j)^2}{\sum (\theta_j - \bar{\theta})^2}$$

  • Mean Squared Error (MSE)

    $$\text{MSE}(\theta) = \frac{1}{N}\sum_{j=1}^N(\theta_j - \hat{\theta}_j)^2$$

Item Calibration & Response Matrix

  • Calibration Phase
    • Estimate item parameters $\hat{z}_1, …, \hat{z}_M$ from response matrix
    • Likelihood objective:

    $$\hat{z}_1, ..., \hat{z}_M = \arg\max_{\theta_1,..., \theta_N, z_1, ..., z_M} \frac{1}{N \times M}\sum_{i=1}^N \sum_{j=1}^M \log p(Y_{i, j} \mid \theta_i, z_j)$$

  • Challenges
    • Adding a new item requires responses from many test takers (e.g., $N=30$)

Fisher Information for Adaptive Selection

  • Objective:

    $$j^* = \arg\max_{j \in \mathcal{Q}} \mathcal{I}(\theta_i; z_j) = - \mathbb{E}\left[ \frac{\partial^2 \log p(Y_{i,j} \mid \theta_i, z_j)}{\partial \theta_i^2} \right]$$

Challenges of Traditional Item Calibration

  • Cost Complexity
    • Calibration costs grow linearly with the number of questions/items
    • Recalibration is needed as item banks are refreshed periodically

Amortized Calibration

  • Amortized Calibration
    • Predicts item parameters using a model instead of collecting responses for every new item
    • Reduces calibration cost to constant time
  • Approaches:
    • Plug-in Amortized Calibration
    • Joint Amortized Calibration

Plug-In Amortized Calibration

  • Training a Model
    • Train a model $\phi$ to predict item parameters $\hat{z}$ based on question content
    • Objective:

    $$\phi = \arg\min_\phi \frac{1}{M} \sum_{j=1}^M ||\hat{z}_j - f_\phi \circ f_\omega(q_j) ||_2$$

Joint Amortized Calibration

  • Joint Estimation of $\theta$ and Item Parameters
    • Optimize both the ability $\theta$ and the prediction model $\phi$ simultaneously
    • Likelihood maximization objective:

    $$\phi = \arg\max_{\theta_1, ..., \theta_N, \phi} \frac{1}{N \times M} \sum_{i=1}^N \sum_{j=1}^M \log p(Y_{i,j} \mid \theta_i, f_\phi \circ f_\omega(q_j))$$

Benefits of Amortized Calibration

  • Cost Reduction
    • Reduces the need for recalibration for each new item
    • Uses question content to predict item parameters efficiently
  • Generalization
    • Allows model generalization across multiple datasets
    • Effective for new, unseen datasets without prior calibration

Conditional Item Generation

  • Challenge:
    • A large, diverse item bank is essential for optimal adaptive question selection
  • Solution:
    • Use a question generator conditioned on item difficulty $z_j$
    • Automates item bank replenishment and reduces resource requirements

Conditional Question Generator

  • Two-Stage Training Strategy
    • Stage 1: Supervised fine-tuning (SFT) with LoRA
    • Stage 2: Proximal Policy Optimization (PPO)
  • Reward Function
    • Maximize reward:

    $$r(q \mid z) = -|| f_\phi(q) - z ||$$

Question Generation via PPO

  • Optimization Objective

    $$\mathcal{L}(\psi) = \mathbb{E}_{q \sim \pi_{\psi}} \left[ r(q \mid z) - \beta D_{KL}[\pi_{\psi}(q \mid z) \, || \, \pi_{\psi_{\text{ref}}}(q \mid z)] \right]$$

  • Inference
    • For each query $z$, generate 64 candidate questions and select the best match
